# 传播宣发计划与引爆链路

> 这是全案的"传播大脑"。本篇集中解决两个问题：
> **① 怎么让玩家看懂这个故事**（把加群/投票/留言/集米/村晚这些散点，讲成一条所有人一眼能懂的叙事）
> **② 怎么在传播上引爆**（节奏 + 平台 + 内容矩阵 + 公关/自媒体串联）
> 状态：评审中（v0.1）｜预算与人力交执行，本篇只给打法

---

## 一、先解决"怎么让玩家看懂"——一个故事，一句话

### 1.1 核心问题
活动有很多小点：加群、投城市菜、留言、领假米、Logo箱、村晚、灯箱点亮……
**如果每个点单独讲，玩家会觉得"零散、不知道在干嘛"。** 传播的第一要务，是把它们**串成一个一句话能讲完、一张图能看懂的故事**。

### 1.2 一句话主线（对外统一口径）
> **「全国米饭，一城一桶米，一路聚到深圳，填满米饭联盟，村晚开席。」**

所有渠道、所有物料、所有稿件，**都讲这同一个故事**，只是换平台换说法。不要每个城市各讲各的。

### 1.3 把散点翻译成"玩家视角"的一句话
| 活动机制（内部说法） | 传播话术（玩家视角） |
|---|---|
| 加分部群 | 找到你这座城的米饭组织 |
| 城市菜投票 | 帮你的城市选一道菜，送去深圳开席 |
| 留言墙 | 在墙上留下你的名字，证明你来过 |
| 领假米/Logo箱 | 给你的城市添上一粒米 |
| 巨型灯箱点亮 | 全国的米聚齐了，灯亮了，开饭 |
| 村晚 | 全国米饭在深圳吃同一顿饭 |

> 传播口径原则：**少讲"机制"，多讲"画面和情绪"**。玩家不关心规则，关心"我在其中是谁、我能看到什么"。

### 1.4 一图读懂（必做物料）
做一张 **"全国聚米路线图" 主 KV / 长图**：12城 → 一路汇到深圳 → 灯箱点亮。
这张图是**整个传播的锚**：预热用它讲故事，过程用它更新进度，收官用它做对比。

## 二、怎么引爆——三阶段传播节奏

> 配合活动两段式（巡游集米 → 深圳村晚），传播分三阶段。时间轴以村晚 12/05 为 T 点倒推。

```
预热聚米期   →   城市接力发酵期   →   深圳村晚引爆期   →   长尾收束
(T-数周)         (各城市站期间)        (12/05当晚±)        (村晚后1-2周)
讲故事、招募      城市接力、攒势能       点亮+开席、刷屏       回收、沉淀、为来年铺垫
```

### 阶段一：预热聚米期（讲故事 + 招募报名）
**目标**：让玩家看懂"要发生一件什么事"，并完成城市站到场/村晚报名的导流。

| 动作 | 内容 |
|---|---|
| 主线官宣 | 官方账号发"全国聚米→深圳开席"主KV + 一句话故事，定调 |
| 一图读懂 | 路线图长图/H5，讲清玩家能参与什么、怎么参与 |
| 城市召集 | 各城分部群/同城话术：本城集米站时间地点 + "帮城市选菜"钩子 |
| 村晚报名 | 放出深圳村晚报名入口（报名筛选制，制造稀缺感："不是谁都能上桌"） |
| 悬念预告 | 浅预告（呼应招新长图），只露故事不露细节，留钩子 |

### 阶段二：城市接力发酵期（攒势能 · 最容易被忽视但最关键）
**目标**：把12个城市站的零散活动，**串成"全国正在接力"的连续叙事**，让没参加的人也觉得"这事在全国发生"。

| 动作 | 内容 |
|---|---|
| 城市接力海报 | 每完成一城，发一张"X城已聚米"接力图，**进度可视化**（已点亮X/12城） |
| 城市色打卡 | 各城用城市代表色出片，12城拼成一组色彩矩阵 |
| 玩家UGC回收 | 鼓励玩家发成员证晒图/留言墙打卡/Logo箱合影，官方转发精选 |
| 城市菜悬念 | "X城选了什么菜？村晚揭晓"——把投票结果**留到村晚**，制造追看动力 |
| 进度同步 | 用"一图读懂"那张图持续更新进度，让故事有"正在进行"的实时感 |

> 关键打法：**接力感 > 单场声量**。单个城市站可能不大，但"12城正在一路聚过来"的连续叙事，比任何一场单独活动都更有势能。

### 阶段三：深圳村晚引爆期（12/05 当晚 · 集中爆破）
**目标**：当晚集中产出刷屏内容，把三阶段攒的势能一次性释放。

| 动作 | 内容 |
|---|---|
| 点亮即发 | 灯箱点亮瞬间，官方第一时间发图/短视频（当晚主传播画面） |
| 三大爆点切片 | 围绕 **点亮 / coser上菜互动 / 全员合影** 出短视频，主攻抖音视频号 |
| 现场UGC放大 | 现场设晒图引导/话题物料，玩家自发产内容，官方实时转发 |
| 直播/视频号同步 | 建议村晚设线上同步，让到不了现场的全国米饭也能"云吃席" |
| 城市菜揭晓集锦 | 12城选了什么菜，做成揭晓合集，城市梗+反差 |

### 阶段四：长尾收束（村晚后 1-2 周）
**目标**：把热闹沉淀成品牌资产，并为下一年/招新埋种子。

| 动作 | 内容 |
|---|---|
| 收官长图/视频 | "从一粒米到一场席"全程回顾，对比开篇路线图，故事闭环 |
| 数据海报 | 12城 · X位米饭 · X条留言 · 一路聚到深圳（用真实数据，不夸大） |
| 玩家故事 | 精选几个玩家/分部长的参与故事，做有人味的UGC/软文 |
| 招新承接 | 把"想加入米饭联盟"的新流量，导向常态化分部社群（呼应招新预告） |

## 三、平台差异化打法

> 同一个故事，不同平台不同讲法。核心原则：小红书讲"我参与的精致感"，抖音讲"现场有多好玩"，微博做"话题广场"，公众号/视频号做"完整叙事"。

| 平台 | 角色 | 内容形态 | 重点 |
|---|---|---|---|
| **抖音** | 引爆主阵地 | 短视频切片：点亮、coser上菜、玩家整活、城市喊话 | 动作强、情绪快、3秒抓人；村晚当晚主战场 |
| **小红书** | 口碑/种草 | 图文：成员证晒图、城市站打卡、留言墙、村晚出片repo | "我参与过"的精致感、攻略感；标题钩子 |
| **微博** | 话题广场 | 话题主持 + 进度接力 + UGC聚合 + 转发抽奖 | 做一个总话题 #全国聚米开席# 沉淀所有声量 |
| **微信公众号** | 完整叙事 | 长文：故事全貌、城市接力纪实、村晚收官稿 | 把故事讲完整，供转发/沉淀/对外 |
| **视频号** | 私域+直播 | 村晚直播、收官回顾视频 | 私域触达、云吃席、长尾 |
| **分部社群（微信群）** | 转化承接 | 全程通知、报名、UGC征集、玩家组织 | 真正的转化和沉淀阵地 |

> 参考标题感（小红书/抖音）：
> - 《我帮我的城市，往深圳送了一道菜》
> - 《全国米饭一路聚到深圳，今晚灯亮了》
> - 《这不是漫展，是全国米饭在深圳吃同一顿饭》
> - 《coser在村晚给我上菜，还喊我饭搭子》

## 四、公关稿 + 自媒体内容矩阵（串联说明）

> 这套矩阵的作用：**用稿件把整个故事"串起来、说明白"**，既对外（行业/媒体），也对内（玩家/社群）。分阶段配齐。

| 阶段 | 稿件类型 | 用途 | 投放渠道 |
|---|---|---|---|
| 预热 | **官方主线通稿** | 定调"全国聚米→深圳开席是什么、为什么做" | 官号公众号 + 媒体 |
| 预热 | 一图读懂/H5 | 让玩家一眼看懂怎么参与 | 全平台 |
| 接力 | **城市站纪实软文**（可每城/可合集） | 记录各城热闹，攒"全国都在参与"的证据 | 公众号 + 小红书 |
| 接力 | 玩家UGC精选合集 | 用玩家自己的内容证明真实热度 | 微博 + 小红书 |
| 引爆 | **村晚现场通稿** | 当晚/次日发，讲点亮+开席的高光 | 官号 + 媒体 + 全平台 |
| 引爆 | 短视频切片矩阵 | 三大爆点分发 | 抖音 + 视频号 |
| 收束 | **收官回顾稿**（图文+视频） | "从一粒米到一场席"故事闭环，沉淀品牌资产 | 公众号 + 视频号 + 媒体 |
| 收束 | 玩家故事稿 | 有人味的个体叙事，强化情感归属 | 小红书 + 公众号 |

**串联逻辑（一条线）：**
```
主线通稿(讲清是什么) → 一图读懂(教怎么参与) → 城市纪实(证明在发生)
→ 村晚通稿(高光引爆) → 收官稿(故事闭环) → 玩家故事(情感沉淀)
```
每一篇都服务同一个主线故事，**不另起炉灶、不各讲各的**。

## 五、UGC 弹药化（让玩家替我们传播）

活动里天然有几样"为晒而生"的素材，要主动设计成**好拍、好发、有理由发**：

| UGC弹药 | 为什么玩家愿意发 | 官方怎么助推 |
|---|---|---|
| 成员证 | 可秀身份、好看、有归属 | 设计要够好看；引导"晒出你的分部成员证" |
| 留言墙 | 找到自己的留言=被看见 | 现场好拍；线上做"全国留言墙"可检索 |
| 城市色Logo箱 | 主视觉、出片 | 箱子做成拍照主体；提供拍照角度引导 |
| 城市菜揭晓 | 城市梗、地域认同 | "我们城选了X菜"激发地域讨论 |
| coser上菜/整活 | 反差、好笑、二次元 | 现场强出片点；鼓励玩家拍coser互动 |
| 灯箱点亮 | 仪式感、情绪峰值 | 当晚引导全场同框拍点亮 |

> 黑话系统在这里**轻量使用**：作为传播用词/话题点缀（如"开饭了""饭搭子""添一粒米"），让内容有圈层味，但**不强造概念、不让玩家费解**。

## 七、合规提示（广告法红线）

- 全程**规避极限词**：不用"最/第一/国家级/唯一/顶级"等表述
- 数据只用**真实可核**的数字（到场/留言/城市数），不编造、不夸大转化
- 玩家信息收集（报名/进群）遵循**个人信息保护**要求，明确用途、不滥用
- 物证措辞守住：成员证是**纪念周边**，**不暗示官方认证或特权**
